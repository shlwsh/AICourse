# MyGit 简化版使用指南

## 简介

MyGit 简化版是一个轻量级的 AI Git 提交工具，可以自动分析代码变更并使用 AI 生成提交信息。

## 功能特点

- ✅ 自动检测代码变更
- ✅ 使用 AI 生成中文提交信息
- ✅ 自动执行 git add 和 git commit
- ✅ 简化流程，无需交互确认
- ✅ 跳过 pre-commit hooks（使用 --no-verify）
- ✅ 轻量级实现，代码简洁

## 使用方法

### 1. 配置 API 密钥

确保 `.env.mygit` 文件已配置：

```bash
DASHSCOPE_API_KEY=your-api-key-here
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DASHSCOPE_MODEL=deepseek-v3
```

### 2. 执行提交

在项目根目录执行：

```bash
bun run mygit
```

### 3. 工作流程

工具会自动完成以下步骤：

1. 检查当前目录是否为 Git 仓库
2. 获取所有代码变更
3. 使用 AI 生成提交信息
4. 执行 `git add .`
5. 执行 `git commit -m "AI生成的提交信息"`

## 使用场景

简化版适合以下场景：

- 快速提交小改动
- 自动化脚本中使用
- 不需要交互确认的场景
- 追求简洁高效的开发流程

如果需要更多控制和交互功能，请使用完整版：

```bash
bun run mygit:full
```

## 注意事项

1. 工具会自动执行 `git add .`，请确保不要提交不需要的文件
2. 建议在使用前先检查 `.gitignore` 配置
3. 如果没有代码变更，工具会自动退出
4. 提交信息由 AI 生成，建议定期检查提交历史

## 故障排除

### 配置文件加载失败

确保 `.env.mygit` 文件存在且格式正确。

### API 请求失败

检查：
- API 密钥是否正确
- 网络连接是否正常
- API 配额是否充足

### Git 命令执行失败

确保：
- 当前目录是 Git 仓库
- 有足够的权限执行 Git 命令
- Git 已正确安装
